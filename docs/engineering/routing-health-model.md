# 路由健康模型

本文档描述 Metapi 路由引擎的站点健康评估策略。

## 设计原则

路由健康度**纯粹由错误信号驱动**。延迟指标仅用于 WebUI 展示和人工观测，不参与路由权重计算。

### 为什么不用延迟？

Metapi 的典型使用场景是聚合 30+ 个公益/中转代理站点。这些站点的延迟特征使得延迟惩罚容易产生误判：

- **思考型模型**（o1、Claude Reasoning）首字延迟可达 30-120 秒，是模型特性而非站点慢
- **非流式请求**（embeddings、images）总延迟取决于 payload 大小
- **模型混用**：同一个站点同时服务 gpt-4o-mini（快）和 o1-pro（慢），全局延迟指标必然误导
- **中转站延迟透传**：公益站大多是 New-API/One-API 风格的纯转发，延迟主要由上游决定

因此，本系统选择了更稳健的策略：**只要不出错就算健康。**

## 健康评估架构

### 两层评估

#### 1. 运行时健康（实时、内存中）

每个站点（+每个模型）维护一个 `SiteRuntimeHealthState`：

| 字段 | 含义 |
|------|------|
| `penaltyScore` | 累积的错误惩罚分数，半衰期 10 分钟自动衰减 |
| `breakerLevel` | 熔断等级（0-3），连续瞬态错误触发升级 |
| `breakerUntilMs` | 熔断解除时间戳 |
| `latencyEmaMs` | 延迟 EMA（**仅观测**） |

**运行时权重乘数计算**（纯错误驱动）：

```
if 熔断中 → multiplier = 0.02（近零）
else → multiplier = 1 / (1 + penaltyScore)
```

不引用 `latencyEmaMs`。

#### 2. 历史健康（持久化，基于 DB 统计）

从路由通道表（`route_channels`）聚合每个站点下所有通道的 `successCount / failCount`：

```
successRate = successCount / totalCalls
multiplier = 1 - ((1 - successRate) × 0.55 × sampleFactor)
```

同样不引用历史平均延迟。

### 最终权重计算

```
finalWeight = baseContribution / sameChannelCount  // 同站点通道均分
  × combinedSiteWeight        // 站点配置权重 × 下游策略倍率
  × runtimeHealthMultiplier   // 运行时错误惩罚乘数
  × historicalMultiplier      // 历史成功率乘数
```

## 错误信号分类

| 错误类型 | Penalty | 说明 |
|----------|---------|------|
| 401/403（凭证失效） | 4.0 | 站点配置问题 |
| 402（额度耗尽） | 5.0 | 号池见底 |
| 429（限流） | 常规 | 触发瞬态熔断 |
| 5xx（服务器错误） | 常规 | 触发瞬态熔断 |
| 连接失败/超时 | 常规 | 网络层问题 |
| quota exhausted / 无可用渠道 | — | 识别为瞬态错误，触发熔断 |

### 熔断机制

连续 3 次瞬态错误触发熔断升级：

| 等级 | 冷却时间 |
|------|----------|
| 0 | 无 |
| 1 | 3 分钟 |
| 2 | 15 分钟 |
| 3 | 60 分钟 |

### 恢复机制

成功请求触发渐进恢复：
- `penaltyScore × 0.5`（减半）
- `breakerLevel - 1`（降一级）

## WebUI 可视化

在路由管理页面（`/routes`），每个通道的决策快照中显示运行时健康标签：

| 标签 | 颜色 | 含义 |
|------|------|------|
| 🔴 站点熔断 / 模型熔断 | 红色 | 熔断器已触发，暂时不被选中 |
| ⚠ 降权 ×0.xx | 橙色 | 错误惩罚导致权重显著降低（combinedMultiplier < 0.5） |
| ⏱ 延迟 Nms | 灰色 | 延迟 EMA 观测值（**仅展示，不参与路由**） |

查看方式：进入路由管理 → 展开某个路由 → 查看各通道右侧的 badge。

## 与上游（原版 metapi）的差异

| 特性 | 原版 | 本 Fork |
|------|------|---------|
| 运行时延迟惩罚 | 有（基线 2.5s，最大 35%） | **已移除** |
| 历史延迟惩罚 | 有（基线 2s，最大 18%） | **已移除** |
| 延迟 EMA 记录 | 有 | 保留（仅观测） |
| 熔断窗口 | 1m / 5m / 30m | 3m / 15m / 60m |
| 失败退避基数 | 15s | 60s |
| 健康 WebUI 可视化 | 无 | ✅ badge + tooltip |
